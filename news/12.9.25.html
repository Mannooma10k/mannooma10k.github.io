MINECRAFT’S NEW VERSION NUMBERING SYSTEM

A small change for clarity.

Written By: Mannooma  |   Published: 12/9/25

In the beginning, there was version **1.0**.
Then came 1.1, and 1.2, and 1.3, and 1.4 –
you can see where this was going. Minecraft’s version numbers went up incrementally with each major update until they hit **1.21** –
the Tricky Trials update – back in June 2024. A lot happened that year,
including the announcement that instead of providing one free update during the year, they wanted to release content more frequently,
called **game drops**.


But what did that mean for version numbering? For a while, it meant they stuck with 1.21. When The Garden Awakens released in December 2024, it was version **1.21.50** on Bedrock Edition, and version **1.21.4** on Java Edition. Then came Spring to Life and Chase the Skies. With Mounts of Mayhem coming soon, the developers felt it was time to update how they version Minecraft releases to reflect their new game drops strategy.


A version number tells you which specific release of Minecraft you are playing. Minecraft prompts you to update to the latest version on both Java and Bedrock Edition by default.

The Change They Are Making

Going forwards, they will be numbering versions based on the **year**. So, for both Bedrock and Java Edition, version numbers in the year **2026** will begin with ‘26’!

Now, Bedrock and Java have different release cadences, so the numbers that follow “26” will be different. There are a few reasons for this (e.g. platform limitations, and the fact that Bedrock has more frequent releases throughout the year) but the **shared prefix** will make it easier for creators (and players) to know which release is which.

The Difference Between Version Numbering in Bedrock and Java Edition

While Bedrock and Java will share the same version number (based on the year), they will still differ slightly based on the release number. This is due to a few factors, including platform limitations and the different release frequencies on Bedrock and Java.

Here’s an example based on **2025**. In 2025, they released three game drops (Spring to Life, Chase the Skies, and The Copper Age) and they have one more to come – Mounts of Mayhem! By applying the new version numbering system to 2025 as an example, players can get a better idea of where Bedrock and Java are the same, and where they are different.

In both Bedrock and Java, the version number will be based on the year – and it will be followed by the **release number**, and then the **patch or hotfix number**. Players can see this in the example above, and they can also see how Java applies the “patch” number as a separate number, whereas Bedrock counts upwards from the release number itself.

There will be some small changes to how they name snapshot versions on Java, too. Moving forwards, they will include the intended version in the names, so it’ll be easier than ever to see which snapshot contains features for which drop. E.g., the first snapshot for Mounts of Mayhem was 25w41a but would have been **25.4-snapshot-1** in this new system.

What Does This Mean for Players?

This change in their version numbering won’t have a huge impact on players. The developers are, however, hoping it’ll make it easier for **creator partners and modders** to understand which of their version numbers represent a game drop, and which of them represent patches or bug fixes to those drops.

They also hope it’ll make it easier for the community to talk about game drops before they have a name! Before the shift to game drops, the community could use the next version number to talk about what was to come. With this new system, players will hopefully be able to do that again!

When Is This Happening?

Mojang will be using this new version numbering system for all of their releases starting **next year** – and players will see it in previews for those versions starting this week!
